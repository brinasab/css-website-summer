1. Intro to For Loops

What Are Control Structures?

Control structures let us write code that can:

  • Make decisions → with conditional statements like if, if...else, etc.

  • Repeat actions → with loops like for, while

Conditional statements and loops are control structures. Last lecture we focused on conditional statements. Today we focus on loops (specifically for loops), and we continue next lecture.

Definition of For Loops

For loops let you repeat an action for each element of an object, usually a vector or list.

Syntax:

for (item in sequence of items) {
  action to be repeated for each item   
}

Example:

for (item in c(1,2,3)) {
  print(item)
}

What’s the repeated action here? How many times? What’s item?

Nested For Loops

In nested for loops the outer loop controls how many times the entire inner loop runs from start to end:

for (i in c(1,2,3)) {
  print(i)
  for (j in c("cat", "dog", "squirrel", "rabbit")) {
    print(j)
  }
}

What’s going on here?

  • The outer loop runs 3 times

  • For each execution of the outer loop, the inner loop runs 4 times

Accumulating a Running Total

Loops are very useful for tracking running totals: adding up scores, votes, or anything else one at a time, and tracking how the total grows with each loop.

This is the same code as the previous slide, but simplified. I removed the extra print statements so we focus on one thing: how my_sum updates after each pass through the loop. Run the code:

my_sum <- 0

for (i in c(1, 2, 3)) {
  my_sum <- my_sum + i
  print(my_sum)
}

my_sum

The problem is that this code prints the updated sum value each time, BUT only saves the final value!

Saving Results from a For Loop

To save all intermediate sum values instead of just the last one, we need to store them in a vector (here output) with one slot per loop iteration. Like this:

my_sum <- 0
output <- vector(mode = "integer", length = 3)  # new!

for (i in c(1,2,3)) {
  my_sum <- my_sum + i
  print(my_sum)
  output[i] <- my_sum     # new!
}

output

output[i] <- my_sum

When i = 1, my_sum = 0 + 1 = 1 → store 1 in output[1] → output becomes: 1 0 0
When i = 2, my_sum = 1 + 2 = 3 → store 3 in output[2] → output becomes: 1 3 0
When i = 3, my_sum = 3 + 3 = 6 → store 6 in output[3] → output becomes: 1 3 6

This saves the running total at each position i in the vector output!

Saving Results from a For Loop

Our current code works but it’s NOT YET optimal:

  • It only behaves correctly because the looping input vector is exactly c(1, 2, 3)
  • If the values in that vector change, the current code may produce incorrect or unexpected results

3. Looping Over Elements vs. Indexes

Elements vs. Indexes: Why It Matters

Run these examples and spot the differences:

# Version A: Looping over elements/values
x <- c(1, 4, 6)
for (i in x) {
  print(i) }    # value

# Version B: Looping over indexes
x <- c(1, 4, 6)
for (i in 1:length(x)) {
  print(i)        # index
  print(x[i]) }   # value at that index

# Same as Version B just using seq_along (recommended)
x <- c(1, 4, 6)
for (i in seq_along(x)) {
  print(i)
  print(x[i]) }

i and x[i]

i is the loop’s index or “counter” and it just counts positions, so it should always go 1, 2, 3, 4, etc.
x[i] is the element or value of the vector x at position i and can be any value

Elements vs. Indexes: Why It Matters

Let’s see what happens to our previous codes when we save the output instead of just printing:

❌ Version A

x <- c(1, 4, 6)
my_sum <- 0
output <- vector(mode = "integer", 
                 length = length(x))

# looping over elements
for (i in x) {
  print(paste("Current i is:", i))
  my_sum <- my_sum + i
  print(paste("Current sum is:", my_sum))
  output[i] <- my_sum
}

output

✅ Version B

x <- c(1, 4, 6)
my_sum <- 0
output <- vector(mode = "integer", 
                 length = length(x))

# looping over indexes
for (i in seq_along(x)) {
  print(paste("Current i is:", i))
  my_sum <- my_sum + x[i]
  print(paste("Current sum is:", my_sum))
  output[i] <- my_sum
}

output

Best Practice: Loop Over Indexes

Tip

Loop over indexes, using for (i in seq_along(x), instead of looping over elements/values.

Advantages of looping over indexes:

  • You can modify, save, or assign values by position (e.g., output[i] <- ...) without writing extra code
  • Your loop works regardless of the values in the vector x
  • You have full control over both: the position i and the value at that position x[i]
  • Looping over indexes followw R best practices

4. From Last Lecture: if / else + For Loop

This is code from our last lecture:

numbers <- c(10, 6, 7)
ifelse(numbers %% 2 == 1, 
       "odd",
       "even")

We can try to rewrite it using if-else statement, but we will get an error (recall why?):

numbers <- c(10, 6, 7)
if (numbers %% 2 == 1) { 
  print("odd")
} else {
  print("even") }

But now we know how to fix that error: with a for loop! This way, the if-else runs one element at a time:

numbers <- c(10, 6, 7)
for (i in seq_along(numbers)) {
  #print(i)
  #print(numbers[i])
  if (numbers[i] %% 2 == 1) {
    print("odd")
  } else {
    print("even")
  } }

5. Using For Loops with Data frames

For Loop in Data frames

So far we’ve looped over vectors, one element at a time.

What about looping over data frames?

A data frame can be looped over row by row or column by column, and both are possible in R.

But since a data frame is really a collection of vectors (each variable is one), R defaults to looping over columns, so that’s what we’ll do here.

Observe the same task with and without a loop

Let’s go back to the penguins:

Our task is to calculate the mean value of several columns in this data frame. We show how to do this without a for loop and with a for loop.


Calculate the mean value of several columns without a loop

We can take the mean() function, and apply it to each column:

penguins |> summarize(avg_bill_length = mean(bill_length_mm, na.rm = TRUE))
penguins |> summarize(avg_bill_depth = mean(bill_depth_mm, na.rm = TRUE))
penguins |> summarize(avg_flipper_length = mean(flipper_length_mm, na.rm = TRUE))
penguins |> summarize(avg_body_mass = mean(body_mass_g, na.rm = TRUE))

This works, but requires a lot of copy/paste! How can we do the same thing with a for loop?

Automate the same task with a loop

First, initialize an empty vector to store the results.

Second, use a for loop to calculate the mean of each (numeric) column of this dataframe:

output <- vector(mode = "double", length = ncol(penguins))

for (i in seq_along(penguins)) {
  #print(i)
  #print(penguins[i])
  output[i] <- mean(penguins[[i]], na.rm = TRUE)
}

output

Let’s examine this code…

  • i will index each column
  • output[i] saves the results in the output vector at the correct position
  • penguins[[i]] ` pulls out the column’s values as a plain vector

Why output[i] and penguins[[i]]?

output <- vector(mode = "double", length = ncol(penguins))

for (i in seq_along(penguins)) {
  output[i] <- mean(penguins[[i]], na.rm = TRUE)
}

output

output is a vector where we store the loop results:

  • To access elements of a vector, use single square brackets []
  • So output[i] assigns the computed value into the i-th position of that vector

penguins is a dataframe (or tibble):

  • But mean() needs a numeric vector
  • To access a dataframe column as a vector, use double brackets [[]]
  • So penguins[[i]] gets the i-th column of the dataframe, and turns it into a vector

Why output[i] and penguins[[i]]?

is.vector(penguins[[3]])           # TRUE (the raw vector)
is.data.frame(penguins[3])         # TRUE (a 1-col tibble/data frame)

mean(penguins[[3]], na.rm = TRUE)  # works, correct type (vector)
mean(penguins[3], na.rm = TRUE)    # error, wrong type (1-col tibble)


With data frames: [[ ]] returns the column itself as a vector, while [ ] returns a one-column data frame or tibble

With vectors: [] and [[ ]] behave the same, but is good practice to consistently use [] when subsetting vectors


Coming Next Week: Beyond For Loops with Data frames

For loops make iteration explicit and are an important programming tool. You will encounter them in many programming languages.

But R is a vectorized language, built to operate on a whole column at once rather than element by element.

That means it has functions that can do the same job as a for loop, just in a way that fits R’s strengths better. Next week we’ll explore them:

  • across() (dplyr): apply a function to many columns in one line
  • map() (purrr): apply a function to every element of a list or column
  • apply() family (base R): the classic vectorized alternative to for loops
