1. Review of core dplyr verbs

Last lecture we learned four main dplyr functions

Function What it does
select() Chooses (keeps or reorders) specific columns by name
filter() Keeps rows based on conditions
group_by() Groups the data by one or more variables for grouped operations
summarize() Reduces each group to a single row using summary statistics (e.g., mean, sum, n)

Back to our Penguins

Load libraries and data:

library(tidyverse)
library(palmerpenguins)
data(penguins)

Explore data (pick one of these, note we have two new ones):

rbind(head(penguins, 5), tail(penguins, 5))
glimpse(penguins)
str(penguins)
summary(penguins)

select() extracts variables, operates on columns

# select two specific variables using their names
select(penguins, species, body_mass_g)

# select all variables except two using the minus sign
select(penguins, -year, -island)

# select a range of contiguous/adjacent variables
select(penguins, species:bill_length_mm)

# select all variables except a range of contiguous ones
select(penguins, -(species:island))

# select all variables that start with
select(penguins, starts_with("bill"))

# select all variables that end with
select(penguins, ends_with("mm"))

filter() extract observations, operates on rows

# check variable values
levels(penguins$species) 
unique(penguins$species)
count(penguins, species)
penguins |> count(species)

# filter by island and species
filter(penguins, island == "Biscoe", species == "Gentoo")
filter(penguins, (island == "Biscoe" & species == "Gentoo"))

# filter by multiple conditions
filter(penguins, species %in% c("Adelie", "Gentoo"), island != "Biscoe")

group_by() aggregates data for the next function

# group by alone? not that useful
penguins |> 
  group_by(island, sex)

# group by one variable: summarize by island
penguins |> 
  group_by(island) |>
  summarize(body_mass_mean = mean(body_mass_g, na.rm = TRUE))

# group by two variables: summarize by island-sex combinations
penguins |> 
  group_by(island, sex) |>
  summarize(body_mass_mean = mean(body_mass_g, na.rm = TRUE))

# filter, group_by, summarize
penguins |>
  filter(species == "Adelie") |>
  group_by(island, sex) |>
  summarize(body_mass_mean = mean(body_mass_g, na.rm = TRUE))

summarize() creates summary of data

# summarize the full dataset
penguins |> 
  summarize(body_mass_mean = mean(body_mass_g, na.rm = TRUE),
            body_mass_sd = sd(body_mass_g, na.rm = TRUE),
            n = n())  # n = nrow(penguins)

# filter first, then summarize only for one species 
penguins |> 
  filter(species == "Gentoo") |> 
  summarize(
    body_mass_mean = mean(body_mass_g, na.rm = TRUE),
    body_mass_sd = sd(body_mass_g, na.rm = TRUE),
    n = n()  # counting rows
  )

Note: summarize() collapses the data into a smaller summary table. You can use summarize() to calculate many kinds of summary statistics, such as means, standard deviations, counts, minimums, maximums, and more. https://dplyr.tidyverse.org/reference/summarise.html

Combining Verbs with the Pipe

Key point: in practice you rarely use one verb alone but you chain verbs together with the pipe |>.

Some combinations come up again and again:

# combo 1: filter() + group_by() + summarize() -- subset, then summarize by group
penguins |> 
  filter(species != "Gentoo") |> 
  group_by(island) |> 
  summarize(body_mass_mean = mean(body_mass_g, na.rm = TRUE))

# combo 2: select() + filter() + group_by() + summarize() -- narrow down columns first
penguins |> 
  select(species, island, sex, body_mass_g) |> 
  filter(sex == "female") |> 
  group_by(species) |> 
  summarize(body_mass_mean = mean(body_mass_g, na.rm = TRUE))

2. More dplyr verbs

Today we learn more dplyr functions

dplyr verbs group naturally by what they operate on: columns, rows, groups. Here’s how today’s new functions connect to what you already know:

Focus Last time (review) Today (new)
Rows filter() keeps rows by condition arrange() reorders rows by value; distinct() keeps only unique rows
Columns rename() renames columns; relocate() reorders columns mutate() adds or changes column values (as central as the verbs we learned last time); across() applies one operation to many columns at once
Groups group_by() defines groups; summarize() summarizes each group count() (a shortcut for group_by + summarize)

arrange() changes order of observations

# basic idea: arrange() sorts rows by one variable 
# here: largest bill length first
penguins |> 
  arrange(desc(bill_length_mm))

# more complex: arrange() can sort by more than one variable
# here: sort by species first, then by bill length within each species
penguins |>
  select(species, contains("bill")) |>
  filter(species != "Adelie") |> 
  arrange(species, bill_length_mm)

relocate() changes order of variable

# move two variables to the front of the dataset
penguins |> 
  relocate(year, sex)

# move year to a specific position: after island
penguins |> 
  relocate(year, .after = island)

count() counts tot obs. by group

# option 1: count penguins by sex the long way
penguins |> 
  group_by(sex) |> 
  summarize(total = n())

# option 2: do the same with count (shortcut!)
penguins |>
  count(sex)
  
# more complex Option 1: count penguins by island-sex combinations
penguins |> 
  group_by(island, sex) |> 
  summarize(total = n())

# more complex Option 2: do the same thing with count
penguins |> 
  count(island, sex)

distinct() returns unique rows based on given columns

# unique combinations of species and island
penguins |>
  distinct(species, island)

# same but keep all columns in the resulting data frame
penguins |>
  distinct(species, island, .keep_all = TRUE)

Note: returns a dataframe with only unique combinations of your input columns https://dplyr.tidyverse.org/reference/distinct.html

mutate() modify or create new columns

One of the Most-Used Verbs in dplyr

mutate https://dplyr.tidyverse.org/reference/mutate.html is one of the most used dplyr verbs, together with filter, group_by, summarize

# create a new column using existing columns
penguins |> 
  mutate(bill_ratio = bill_length_mm / bill_depth_mm)
  
# create a new column by converting body mass from grams to kilograms
penguins |> 
  mutate(body_mass_kg = body_mass_g / 1000)

across()

Examples with and without across() with numerical variables:

# mean for several variables without across
penguins |>
  summarize(
    mean_bill_length_mm = mean(bill_length_mm, na.rm = TRUE),
    mean_bill_depth_mm = mean(bill_depth_mm, na.rm = TRUE),
    mean_flipper_length_mm = mean(flipper_length_mm, na.rm = TRUE),
    mean_body_mass_g = mean(body_mass_g, na.rm = TRUE)
  )

# same operation with across
penguins |>
  summarize(
    across(
      c(bill_length_mm, bill_depth_mm, flipper_length_mm, body_mass_g), 
      ~ mean(.x, na.rm = TRUE)  # function(x) mean(x, na.rm = TRUE)
    ))

across()

Examples with and without across() with categorical variables:

# number of distinct entries without across
penguins |>
  summarise(distinct_species = n_distinct(species),
            distinct_island = n_distinct(island),
            distinct_year = n_distinct(year))

# number of distinct entries with across
penguins |>
  summarise(across(c(species, island, year), 
                   n_distinct))

# the opposite
penguins |>
  summarise(across(-c(species, island, year), 
                   n_distinct))

# everything
penguins |>
  summarise(across(everything(), 
                   n_distinct))

3. Data Cleaning

3.1 Data Cleaning: Renaming & Recoding Variables

A Few More dplyr Functions

We now introduce a few more dplyr functions: rename(), recode(), and case_when().

We introduce them here, in the context of renaming and recoding variables, because that’s how they’re typically used in practice.

Note

recode() and case_when() only work on a single column’s values, so they’re always used inside mutate()

Definitions of Rename and Recode

Rename: change variable names (column names)

Recode: change values/levels of categorical variables (column values; e.g., inside a column)

Uses of Rename and Recode

When might you need to rename variables or recode their values?

  • You are cleaning data
    • The variable name has issues: Flipper Length (mm) → flipper_length_mm
    • You need to standardize values: "Good" and "GOOD" → "good"
    • Etc.
  • You are preparing data for modeling or visualization
    • You want to recode "FEMALE"/"MALE" to 0/1 for a regression model
    • Etc.

Renaming Variables with rename

To change variable names (column names) the most common way is by using rename()

Change the name of the variable studyName to study_name:

# check before renaming
str(penguins_raw)
penguins_raw %>% select(studyName)

# rename
penguins_raw %>% rename(study_name = studyName)   # new = old

# remember to save to keep changes
p <- penguins_raw %>% rename(study_name = studyName)
p %>% select(study_name)

Recoding Variables Method 1: with mutate + recode

To change variable values (usually levels of categorical variables), we learn two approaches. We take the variable Sex and turn MALE into 1, FEMALE into 0.

Method 1:

# check before recoding
penguins_raw %>% count(Sex)

# mutate + recode
p <- penguins_raw %>%
  mutate(sex = recode(Sex, "MALE" = 1, "FEMALE" = 0))

# compare
penguins_raw %>% count(Sex)
p %>% count(sex)

Recoding Variables Method 2: with mutate + case_when

Method 2:

# mutate + case_when
penguins_raw %>%
  mutate(Sex = case_when(Sex == "MALE" ~ 1,
                         Sex == "FEMALE" ~ 0,
                         TRUE ~ NA_real_))
  
# like for method 1 (previous code) save results to keep changes and compare

Note

With case_when() each logical condition ~ value pair acts like if → then:

  • for each row, R looks if the condition is TRUE: “If you find the value MALE in Sex, then convert it to 1”
  • TRUE ~ NA_real_ tells R: “If no previous condition was met, then return NA as as a number”

The Role of mutate in Recoding

The dplry verb mutate has many uses: create new columns or modify existing columns values.

Note

For recoding, use mutate + recode or mutate + case_when, because the goal is changing the column’s values:

  • Method 1: mutate(Sex = recode(Sex, "MALE" = 1, "FEMALE" = 0))
  • Method 2: mutate(Sex = case_when(sex == "MALE" ~ 1, sex == "FEMALE" ~ 0))

Which method is “best”? It depends, but method 2 is more popular (flexible/powerful)!

Rename vs Recode: Syntax Reference

Function What It Changes Syntax + Example Tips
rename() Column names rename(new_name = old_name)
rename(notes = Comments)		 No quotes around variable names
recode() Column values recode(variable, "old" = new)
recode(Sex, "MALE" = 1)		 Check function doc to see when quotes are needed
case_when() Column values case_when(variable == "old" ~ new)
case_when(Sex == "MALE" ~ 1)		 Check function doc to see when quotes are needed

3.2 Data Cleaning: Syntactic vs. Non-syntactic Variable Names

Syntactic (Valid) Variable Names in R

Valid Names in R:

  • Use letters, numbers, and the symbols . or _
  • But cannot start with a number or symbol

Examples of Valid Names:

flipper_length_mm
flipper.length.mm
flipper.length_mm     # valid but poor style
FlipperLengthMm       # valid but poor style

Non-Syntactic (Invalid) Variable Names in R

What Makes a Name Invalid:

  • Contains spaces or symbols
  • Starts with a number or symbol
  • Uses reserved words (e.g., TRUE, NULL, if, function)
  • Type ?Reserved in the Console for the full list

Examples of Invalid Names:

Flipper Length (mm)
@_flipper_length_mm
flipper_ length_mm
flipper-length-mm
.flipper.length.mm

Tip

For best coding style, use snake_case for all your variables names and keep them to three words maximum. Example: q3_religion

How to Handle Non-syntactic Names, and Why It Matters

You should avoid creating non-syntactic names, BUT you’ll often encounter them, especially in datasets not created in R (Excel or other sources). If you don’t handle them properly, R will throw errors when you try to use them.

What to Do:

1. Use backticks to refer to them (e.g., `Flipper Length (mm)`)
2. Use rename() to change them to syntactic names

Non-syntactic names will break code if you forget to wrape them in backticks, so renaming avoids issues.

Working with Non-syntactic Names in Practice

Imagine you are working on political ideology by country, assembled by someone else. The data are in Excel and when you imported them in R they look like this:

df <- tibble(country = c("Italy", "Germany", "France", "Italy", "United States"),
                  `4 ideology` = c("communism", "fascism", "anarchism", "fascism", "capitalism"))


To use the non-syntactic variable name without changing it and without errors, you must use backticks:

select(df, `4 ideology`)
