1. Alternatives to For Loops in R

Last Lecture: we used a for loop with a dataframe

In lecture 16 we learned how to apply a for loop to calculate mean value of several columns in a dataframe. Like this:

library(tidyverse)
library(palmerpenguins)

output <- vector(mode = "double", length = ncol(penguins))

for (i in seq_along(penguins)) {
  #print(i)
  #print(penguins[i])
  output[i] <- mean(penguins[[i]], na.rm = TRUE)
}

output

Let’s examine this code…

  • i will index each column
  • output[i] saves the results in the output vector at the correct position
  • penguins[[i]] ` pulls out the column’s values as a plain vector

Last Lecture we ended with this slide

For loops make iteration explicit and are an important programming tool. You will encounter them in many programming languages.

But R is a VECTORIZED language, built to operate on a whole dataframe column at once rather than element by element.

That means it has functions that can do the same job as a for loop, just in a way that fits R’s strengths better. Next week we’ll explore them:

  • across() (dplyr): apply a function to many columns in one line
  • map() (purrr): apply a function to every element of a list or column
  • apply() family (base R): the classic vectorized alternative to for loops

Why We Even Learn Loops in R?

Many tasks in R can and should be done without writing a for loop.

So why do we even learn loops in R? Because understanding loops is important:

  • to build logic and control flow skills
  • for non-vectorized tasks**, like row-by-row operations
  • to build more complex workflows
  • to be able to read and understand other people code
  • for general literacy on programming

Even if you don’t use loops every day, knowing how they work will strengthen your skills as coder.

Three Main Alternatives to For Loops in R

When working with dataframes, R provides the following alternatives to for loop:

Function Family Package Best Used For Docs
map_*() purrr Applying functions to elements of lists or columns map docs
across() dplyr Applying the same function to multiple columns inside mutate() or summarize() across docs
*apply() base R Row/column-wise operations on matrices or data frames apply docs


The asterisk * is a placeholder for a family of related functions (e.g., map_dbl(), map_chr(), sapply(), etc.)


Option 1: Replacing for loops with map() functions

The first option to replace a for loop is the family of map_*() functions from the purrr package.

What it does: applies a function to each element of a list or vector (for a data frame: each column), and returns the results in the type of output you choose

Best use: flexible, all-purpose iteration; unlike across(), it does not need to live inside a dplyr verb like summarize() or mutate()

Option 1: Replacing for loops with map() functions

The so-called “map functions” come from the purr package in R: https://purrr.tidyverse.org/reference/map.html

There are several map_*() functions each creates a different type of output:

  • map() makes a list
  • map_lgl() makes a logical vector
  • map_int() makes an integer vector
  • map_dbl() makes a double vector
  • map_chr() makes a character vector

This is the same idea as in the for loop when we specify the mode of our output vector.

Let’s see a few examples using the penguins data.

Option 1: Replacing for loops with map() functions

To use these properly, you need to pick the appropriate map() function (see previous slide) and specify at least two main arguments:

  • what you are iterating over
  • what you are calculating

Here we are calculating the mean of all numeric columns of the penguins dataframe:

penguins |> 
  select(where(is.numeric)) |>
  map_dbl(mean, na.rm = TRUE)

Option 2: Replacing for loops with across()

Another option to replace a for loop is the across() function from dplyr

What it does: applies the same operation (e.g., mean) to multiple columns at once

Best use: since it comes from dplyr, it works seamlessly inside dplyr verbs like mutate() and summarize() which these are its favorite verbs to work with! You might remember this function from Lecture 7

Option 2: Replacing for loops with across()

This example calculates the mean of several columns in a data frame. We used it in Lecture 16, when we rewrote it using a for loop:

penguins |> summarize(avg_bill_length = mean(bill_length_mm, na.rm = TRUE))
penguins |> summarize(avg_bill_depth = mean(bill_depth_mm, na.rm = TRUE))
penguins |> summarize(avg_flipper_length = mean(flipper_length_mm, na.rm = TRUE))
penguins |> summarize(avg_body_mass = mean(body_mass_g, na.rm = TRUE))

Now, we rewrite it again but this time using across()

Option 2: Replacing for loops with across()

Rewrite previous example using across():

penguins |> 
  summarize(
    across(
      .cols = where(is.numeric),           # select columns by type
      .fns = ~ mean(.x, na.rm = TRUE)))    # apply mean, skipping NA


The function across() has two main arguments:

  • .cols: the columns to operate on. You can select them by position, name, or type - in this example, by type using where(is.numeric)

  • .fns: the function, or list of functions, to apply to each column (in this example, we use mean() and .x is a placeholder for the current column being processed)

You can omit .cols and .fns if you pass the arguments in the correct order, like that: penguins |> summarize(across(where(is.numeric), ~ mean(.x, na.rm = TRUE)))


Option 2: Replacing for loops with across()

Tip: you can also add names to the newly computed means to improve readabilty

penguins |> 
  summarize(
    across(
      .cols = where(is.numeric),                   
      .fns = ~ mean(.x, na.rm = TRUE),             
      .names = "avg_{.col}"                   
    )
  )


What does \"avg_{.col}\" mean?

  • "avg_" is a literal prefix that will be added to each new column name.
  • {.col} is a placeholder that will be replaced with the original column name.

So if you’re applying mean() to a column named bill_length_mm, the result will be named avg_bill_length_mm

Option 3: Replacing for loops with apply() functions

Finally, a third option to replace a for loop in R is using one of the apply() functions from base R

What it does: applies the same operation (e.g., mean) across the rows or the columns of a data frame

Best use: when working outside tidyverse verbs (this is from base R!); applying a function across columns is the most common use

Option 3: Replacing for loops with apply() functions

We looked at this example earlier (which calculates the mean of several columns in a data frame) and rewrote it using a for loop. Now we rewrite it again but this time using apply():

penguins |> summarize(avg_bill_length = mean(bill_length_mm, na.rm = TRUE))
penguins |> summarize(avg_bill_depth = mean(bill_depth_mm, na.rm = TRUE))
penguins |> summarize(avg_flipper_length = mean(flipper_length_mm, na.rm = TRUE))

Option 3: Replacing for loops with apply() functions

Rewrite the previous example using apply():

# manually select only numeric column using base R syntax
penguins_numeric <- penguins[ , c("bill_length_mm", "bill_depth_mm", 
                                  "flipper_length_mm", "body_mass_g")]

# apply mean to these columns
apply(penguins_numeric, 2, mean, na.rm = TRUE)

Note the 2 stays for MARGIN = 2 (apply on the columns). If you write 1 it means rows.

Type ?apply in your Console for more info.

Option 3: Replacing for loops with apply() functions

The apply() family refers to several related functions you can use:

Function Use Case Input Type Output Type Why Use It
apply() Apply a function across rows or columns Matrix, data frame Vector or matrix For row/column-wise operations
lapply() Apply a function to each element List, data frame List Keeps outputs as list (safe, flexible)
sapply() Like lapply() but simplifies output List, data frame Vector or matrix Shorter output, easier to read
vapply() Like sapply(), but requires output type List, data frame Vector or matrix Safer than sapply() (explicit type checking)
tapply() Apply a function to vector values by group Vector + Factor(s) Vector For group-wise summaries (e.g., mean by sex)


You don’t need to memorize each one — but remember there are multiple apply() options (like there are multiple map() options). Look them up and pick the one that fits your task!


Option 3: Replacing for loops with apply() functions

# Use numeric columns from penguins dataset
num_data <- penguins[ , 3:6]

# apply(): function across columns (MARGIN = 2)
apply(num_data, 2, mean, na.rm = TRUE)

# lapply(): list output
lapply(num_data, mean, na.rm = TRUE)

# sapply(): simplified vector output
sapply(num_data, mean, na.rm = TRUE)

# vapply(): safe version of sapply()
vapply(num_data, mean, na.rm = TRUE, FUN.VALUE = numeric(1))

Tips for the apply() Family

  • apply() works best with matrix-like structures
  • lapply() is list-safe, but sapply() is more concise
  • vapply() avoids surprises (e.g. unpredictable output type) by enforcing output type (here returns a single numeric value)

2. While Loops

Definition of While Loops

We focused on “for loops” because they are the most common, but R, and many other programming languages, also supports “while loops”

How while loops work:

  • Evaluate a condition
  • If the condition is TRUE, run the loop body
  • Start over: re-evaluate the condition
  • Repeat until the condition turns into FALSE (then the while loop is over)

While Loop Syntax

Syntax:

while (condition to be evaluated) {
  statement(s)
}

Example:

counter <- 1

while(counter <= 4) {
  print(counter)
  counter <- counter + 1
}

While Loop Examples

Take the previous code, but this time print counter also at the end:

counter <- 1

while(counter <= 3) {
  print(counter)
  counter <- counter + 1
  print(counter)
}

Why are the results different from the previous code?

While Loop Examples

Take the previous code, but this time we do not increment our counter variable. What is the output of this code?

counter <- 1
while(counter < 3){
  print(counter)
}

While Loop Examples

Observe this last one. What is the output of this code?

counter <- 1
while(counter < 4){
  print(counter)
  multiply <- counter * 100
  print(multiply)
  counter <- counter + 1
  print(counter)
}

While Loop Uses

While Loops are useful when you don’t know in advance how many times to iterate: you want the loop to continue until a condition is met.

Example use cases:

  • Keep looping until you get three heads in a row from random coin flips
  • Keep accepting user input until you reach a target number of responses

While loops typically require a counter variable: set it outside the loop, and update it inside the loop, otherwise the loop never ends.

While loops are important but less common than for loops, especially for the types of tasks we do in this course.

3. Introduction to Functions

From using to writing functions

Today we learn the following concepts:

  • Using existing functions
  • Writing your own function
  • The return statement
  • stop()

Using existing functions

Functions are all over R! We have been using functions from day 1 of this course. For example:

a <- c(1:5)
a

sum(a)
mean(a)

What is a function?

A function:

  • is a block of code that accomplishes a task

  • has a name that typically describes the task, like sum() or mean()

  • takes one or more inputs arguments), processes them, and returns one or more outputs

Using existing functions

To use a function, we need to know its arguments.

In the console, type help() with the function’s name in parenthesis to learn more about a function behavior and its arguments:

help(sum)
help(mean)

Some functions come from base R, like sum() or mean()

Other functions belong to specific packages, like summarize() or mutate() which belong to dplyr

Writing custom functions

Other than using functions created by others, we can also write our own functions!

To do that we need the following:

  • name: something that describes the task that the function performs
  • arguments (input): at least one data structure (e.g., vector, etc.)
  • body: self-contained lines of code that manipulate the input, defining what the function does
  • return (output): statement that returns the result of the function’s calculations

Syntax:

name <- function(arg1, arg2, ...) { 
    result <- body with code that does things using arg1, arg2, etc.
    return(result)
    }

Writing custom functions

In R we can also write our own function!

For example, we could write our own sum function to calculate the sum of the elements of a numeric vector, like this:

my_sum <- function(vector) {
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  return(total)
}

x <- c(2,1,5,2)
my_sum(x)

Let’s unpack this code…

Writing custom functions

Tip! Write and test the code outside a function to make sure it works, like this:

x <- c(2,1,5,2)

total <- 0
for (i in seq_along(x)) {
  total <- total + x[i]

Then wrap the code into a function and test it with different inputs:

my_sum <- function(vector) {
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  return(total)
}

x <- c(2,1,5,2)
my_sum(vector = x)

y <- c(4,4)
my_sum(y)

Writing custom functions

To use a function, we need to call it with specific values.

In the example from the previous slide, our custom function my_sum() takes one argument, vector. When we call the function, this will be a specific vector, such as x, y, etc.

x <- c(2,1,5,2)
my_sum(vector = x)

y <- c(4,3,7)
my_sum(y)

Advantages of writing custom functions

We wrote a sum function to illustrate how user-written functions work, but typically if there is already a built-in function in R , it’s best to use it (vs. rewriting it)

Instead write your own function when…

  • there is no a built-in function

  • you want to reduce repetitive code and chances for mistakes

  • you want to reuse code

  • you want organize code (e.g., one function imports the data, another cleans the data, etc.)

Tips for writing functions: name

Use unique names:

  • If your function name matches the name of an existing R function, your function will replace the existing function for your current session (e.g., write my_sum, do not re-write sum)
  • Unless there is a specific reason (e.g. learning purposes, etc.), do not create a new function like my_sum() when R already has a built-in sum() function you can use

Use informative names:

  • The name should suggest what the function does
  • It should be short
  • Avoid reserved words like if, else, for, function, etc. To see the full list: type help(reserved) in the Console

Tips for writing functions: variables scope

Variables defined inside a function are not available outside it. Their scope lies within and is limited to the function itself.

Run this in a fresh R session: total exists only inside my_sum():

my_sum <- function(vector) {
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  return(total)
}

x <- c(2,1,5,2)
my_sum(x)
total   # Error: object 'total' not found

The return statement(s)

If you do not write a return statement, your output will be the last statement in your code:

my_sum <- function(vector) {
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  total
}

x <- c(2,1,5,2)
my_sum(x)

This works, but it’s best practice to use return for clarity and readability. Remember that any code after return will be ignored! Let’s see a couple of examples…

Write multiple return statements: Example 1

describe_number <- function(x) {
  if (x == 0) {
    return("x is zero (special case!)")
  }
  if (x > 0) {
    return("x is positive")
  } else {
    return("x is negative")
  }
}

Write multiple return statements: Example 2

my_sum <- function(vector) {
  
  # check 1: input must not be missing
  if (missing(vector)) {
    return("Error: you must provide a vector")
  }
  
  # check 2: input must be numeric
  if (!is.numeric(vector)) {
    return("Error: input must be a numeric vector")
  }
  
  # compute the sum
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  return(total)
}

# examples
my_sum(c(2, 1, 5, 2))
my_sum(c("one", 2, 3))
my_sum()

Write one return statement with multiple outputs

Sometimes you want to return multiple objects and collect them into a list or a vector:

my_sum <- function(vector) {
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  return(list(total, i))
}

x <- c(2,1,5,2)
y <- my_sum(x)

Using stop() when writing a function

Define a function celsius_to_fahr() that converts temperatures from Celsius to Fahrenheit using the formula fahrenheit = (celsius * 9/5) + 32

celsius_to_fahr <- function(celsius) {
  fahr <- (celsius * 9/5) + 32
  return(fahr)
}

celsius_to_fahr(0)
celsius_to_fahr(-20)

Using stop() when writing a function

To ensure this function works properly, the argument must be numeric. Otherwise, the conversion between the two temperature scales will not work.

Thus, we use if-else to verify if the provided argument is numeric. If not, we use stop() to raise an error and stop the function execution!

celsius_to_fahr <- function(celsius) {
  if (!is.numeric(celsius)) {
    stop("error: input must be a number, try again")
  } else {
    fahr <- (celsius * 9/5) + 32
    return(fahr) 
  }
}

celsius_to_fahr("zero")

Question: Based on what you see in this code, what’s the difference between stop() and return()?

Using stop() when writing a function

Most functions exit in one of two ways. They either return() a value, indicating success, or they stop() and throw an error, indicating failure. Therefore:

  • return(): exits the function NORMALLY, handing back a value
  • stop(): exits the function WITH AN ERROR, so execution halts, the console shows Error:, and no misleading value is returned

Using stop() when writing a function: obseve this comparison

Approach 1: using return() for errors ❌

my_sum <- function(vector) {
  
  # input check with return(): the error message is just a regular value
  if (!is.numeric(vector)) {
    return("Error: input must be a numeric vector")
  }
  
  # compute the sum
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  return(total)
}

result <- my_sum(c("one", 2, 3))  # no error shown!
result                            # only now we see something is off
result + 10                       # fails HERE, with a confusing message

Using stop() when writing a function: obseve this comparison

Approach 2: using stop() for errors ✅

my_sum <- function(vector) {
  
  # input check with stop(): raise a real error
  if (!is.numeric(vector)) {
    stop("input must be a numeric vector")
  }
  
  # compute the sum (identical to previous code)
  total <- 0
  for (i in seq_along(vector)) {
    total <- total + vector[i]
  }
  return(total)
}

result <- my_sum(c("one", 2, 3))  # Error: input must be a numeric vector, halts immediately
                                  # with OUR message; result is never created
result + 10                       # in a script, this line is never reached
